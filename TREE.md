# Essays tree

Conceptually, this repository is a galaxy: works can branch, orbit one another, and send questions back across the structure. Operationally, it is a tree so that every text has one clear home.

## What each level means

- **Essays** is the author-level public index. It can eventually contain families other than Trash-End.
- **Trash-End** is the current family and point of origin. It is not assumed to be the permanent center of every future work.
- **Works** have their own README, status, reading path, and source boundaries.
- **Current** contains complete candidates under active comparison.
- **Drafts** contains unfinished editorial material.
- **Sources** contains public-safe source manifests and distilled development records.
- **Legacy** preserves earlier versions without rewriting or inventing history.
- **Project records** contains decisions, provenance, and chronology for the living tree.

## The separate fragment archive

The early web prototype called *Archivio di Transizione* is not another level of this tree. It is a separate historical surface containing four short early-stage texts.

Those texts are classified as **fragments**:

- they preserve the vocabulary, tensions, and direction of an early phase;
- they are not finished essays;
- they are not release candidates;
- they do not define the present conclusions of The Big Question or the wider tree;
- they should be republished only if clearly labeled as fragments or rewritten into later works.

Keeping that archive separate prevents two different kinds of history from collapsing into one another. Legacy records how an individual work changed. Fragments record ideas that appeared before the works had found their final form.

## Public status

No stable essay release has been designated. Current candidates are public review artifacts, not final scientific claims. The root README and each work README are the authoritative entry points; compatibility index files remain secondary.
